# 仕様 (SPECIFICATIONS)

## Config

- `CFG-001`
  - 保存時に `config.json` へ URL、モデル、暗号化済み API key を書き込む。
- `CFG-002`
  - API key の平文は `config.json` に残さない。
- `CFG-003`
  - 次回起動時に URL、モデル、API key を復元できる。
- `CFG-004`
  - `config.key` 欠損時は URL とモデルだけを復元し、API key は空欄と警告にする。
- `CFG-005`
  - `config.key` 破損時もアプリは継続し、非秘密情報だけを復元する。
- `CFG-006`
  - 旧 AEAD ラベルと旧 XOR 形式の保存データを互換復元する。
- `CFG-007`
  - 壊れた `config.json` は parse error として扱う。
- `CFG-008`
  - Config 画面では `Timeout`（秒）、`TextScale`（文字数スケール）、`Freq`（Hz）、`SoundEnabled`（true/false）を編集できる。
- `CFG-009`
  - `SoundEnabled` フィールドではスペースキーでトグル操作を行う。
- `CFG-010`
  - ゲーム設定は `String` 型として保存し、`config.json` の `game` セクションに書き込む（旧ファイルは `#[serde(default)]` で読み込み可能）。
- `CFG-011`
  - BGM は起動時の設定を使用し、再起動後に反映される。キープレス音は設定変更後即時反映される。
- `CFG-012`
  - `Config` 画面では、現在フォーカスしている入力欄の現在入力位置にカーソルを表示する。
- `CFG-013`
  - `Config` 画面の文字列入力欄では、`Left` / `Right` で入力位置を 1 文字ずつ移動する。
- `CFG-014`
  - `Config` 画面の文字入力は現在入力位置へ挿入し、挿入後のカーソルは挿入文字の後ろへ移動する。
- `CFG-015`
  - `Config` 画面の `Backspace` は現在入力位置の直前の文字を削除し、削除後のカーソルは 1 文字左へ移動する。
- `CFG-016`
  - `Config` 画面の API key 欄は平文を表示せず、実文字数と同じ長さの `*` で表示する。
- `CFG-017`
  - `SoundEnabled` は文字列入力欄ではないため、`Left` / `Right`、文字入力、`Backspace` で値を変更しない。

## Runtime

- `RUN-001`
  - タイトル画面では `Start Game`、`Practice Mode`、`Stats`、`Config` を常に上下キーで巡回選択できる。
- `RUN-002`
  - `Start Game` 選択時は `Local` 生成で `Loading` に遷移し、生成完了で `Typing` に進む。
- `RUN-003`
  - `Practice Mode` 選択時は `Local` 生成かつタイムアウト無効で `Loading` に遷移し、生成完了で `Typing` に進む。
- `RUN-004`
  - `Start Game via Google AI Studio` は Google AI Studio の `API URL`、`API Key`、`Model` がすべて空でない場合だけ表示し、選択時は `Google AI Studio` 生成で `Loading` に遷移し、生成完了で `Typing` に進む。
- `RUN-005`
  - `Start Game via GroqCloud` は Groq の `API URL`、`API Key`、`Model` がすべて空でない場合だけ表示し、選択時は `Groq` 生成で `Loading` に遷移し、生成完了で `Typing` に進む。
- `RUN-006`
  - `Stats` 選択時は保存済み履歴の統計サマリ画面に遷移する。
- `RUN-007`
  - 生成失敗時は `Menu` に戻り、失敗理由を表示する。
- `RUN-008`
  - `Loading` 中に `Esc` を押すと生成要求を破棄して `Menu` に戻る。
- `RUN-009`
  - `Config` 画面で `Enter` は保存、`Esc` は破棄、`Backspace` はカーソル直前の文字削除として扱う。
- `RUN-010`
  - `Typing` 中に全文入力またはタイムアウトで `Result` に遷移する。
- `RUN-011`
  - Config で `timeout=0` に設定して開始した `Typing` は、全文入力時のみ `Result` に遷移し、タイムアウトでは終了しない。
- `RUN-012`
  - `Practice Mode` で開始した `Typing` は、全文入力時のみ `Result` に遷移する。
- `RUN-013`
  - `Typing` は strict 判定を行い、入力文字がターゲット文字と一致した場合のみ入力位置を進める。
- `RUN-014`
  - `Typing` 中に `Esc` を押すと `Menu` に戻る。
- `RUN-015`
  - `Result` 画面は `Typed`、`Misses`、`Accuracy`、`Time`、`WPM` を表示する。
- `RUN-016`
  - `Typed` は `Backspace` で減らない総入力数 `total_typed_count` を表示する。
- `RUN-017`
  - `Accuracy` は `accuracy = (total_typed_count - incorrects) / total_typed_count * 100` で算出し、小数 1 桁で表示する。
- `RUN-018`
  - `total_typed_count = 0` のまま `Result` に遷移した場合、`Accuracy` は `0.0%` と表示する。
- `RUN-019`
  - `Misses` は既存の `incorrects` をそのまま表示し、ミス修正回数や実ミス数の分離は今回の仕様に含めない。
- `RUN-020`
  - `Typing` 画面は現在 WPM の数値表示に加えて、`ratatui` の線グラフ系ウィジェットによる WPM 推移グラフを表示する。
- `RUN-021`
  - `Result` 画面は `Typed`、`Misses`、`Accuracy`、`Time`、`WPM` に加えて、セッション終了時点の WPM 推移グラフを表示する。
- `RUN-022`
  - WPM 推移グラフは各画面の本文領域と別座標に配置し、本文テキストと重ならない。
- `RUN-023`
  - WPM 推移サンプルはタイピング開始時に空で初期化し、タイピング中の再描画に合わせて更新する。
- `RUN-024`
  - `Result` 画面の WPM グラフは、Typing 中に蓄積した `wpm_history` の最終内容をそのまま表示し、結果画面専用の別履歴は持たない。
- `RUN-025`
  - 画面サイズが不足する場合でも、各画面の主要情報の可読性を優先し、グラフは縮小または簡略表示しても重なりを起こさない。
- `RUN-026`
  - `src/presentation/ui/render/typing.rs` と `src/presentation/ui/render/result.rs` を含む描画モジュールでは、`#[cfg(test)] mod tests` を通常関数より後ろ、すなわちファイル末尾へ配置し、`clippy::items_after_test_module` を発生させない。
- `RUN-027`
  - `Typing` 画面では、出題文字列上へ端末カーソルを表示しない。
- `RUN-028`
  - 現在位置表示の変更は `Typing` 画面に限定し、`Menu`、`Config`、`Loading`、`Result` の表示には影響させない。
- `RUN-029`
  - `Typing` 画面の現在入力すべき文字は `Yellow + Bold` で表示し、下線は付けない。
- `RUN-030`
  - `Typing` 画面の未来の未入力文字は `Gray` で表示する。
- `RUN-031`
  - `Typing` 画面の正答済み文字は `Green`、誤入力位置は `White` on `Red` background で表示する。
- `RUN-032`
  - `Typing` 画面では、出題文字列が改行で折り返されても現在入力すべき文字の強調が表示上の現在位置と一致する。
- `RUN-033`
  - `Typing` 画面の WPM グラフは、セッション内で高い WPM に相当する線分をオレンジで強調し、それ以外は既定の緑系配色を使う。
- `RUN-034`
  - `Typing` 中の WPM 履歴は、入力操作が発生していない区間でも最後の入力から 2 秒の猶予までは直前の WPM 推移を維持し、猶予経過後に 0 として記録する。
- `RUN-035`
  - Timed セッション完了時は WPM、Accuracy、Miss count、入力時間、生成元、モード、正解側ミス文字を `history.json` へ追記する。
- `RUN-036`
  - Practice Mode 完了時は `history.json` へ追記しない。
- `RUN-037`
  - Result 画面は保存済み履歴から自己ベスト WPM、平均 WPM、平均正確率、直近10回の WPM 推移、頻出ミス文字を表示する。
- `RUN-038`
  - Stats 画面は保存済み履歴から自己ベスト WPM、平均 WPM、平均正確率、直近10回の WPM 推移、頻出ミス文字を表示する。
- `RUN-039`
  - Stats 画面で `Enter` または `Esc` を押すと `Menu` に戻る。
- `RUN-040`
  - `Menu` 画面の選択中項目は、項目ラベルの左に `▶︎` と空白を付けて表示する。
- `RUN-041`
  - `Menu` 画面の非選択項目は、項目ラベルの左に空白と空白を付けて表示し、ラベル開始位置を選択項目と揃える。
- `RUN-042`
  - 選択マーカーの変更は描画層に閉じ、`MenuItem`、選択順、キー入力処理、生成元選択、`Stats` / `Config` 遷移は変更しない。
- `RUN-043`
  - `Typing` 画面の `Target Text` ブロックは、出題本文の前に2行の空行を描画する。
- `RUN-044`
  - `Typing` 画面の `Target Text` ブロックは、出題本文の後に2行の空行を描画する。
- `RUN-045`
  - 出題本文が折り返しで複数行になる場合、`Target Text` ブロックは折り返し後の本文全体の後に2行の空行を描画する。
- `RUN-046`
  - `Target Text` ブロック余白変更は `src/presentation/ui/render/typing.rs` の描画層に限定し、入力処理、WPM 履歴、ヘッダー、フッター、Result / Stats 画面は変更しない。
- `RUN-047`
  - `Typing` 画面で WPM グラフを表示する場合、`Target Text` ブロックには枠線込みで最低8行を割り当て、内側に本文前2行、本文2行、本文後2行を表示できる高さを確保する。
- `RUN-048`
  - `Typing` 画面の高さが不足し、WPM グラフと `Target Text` の最低高さを両立できない場合は、WPM グラフより `Target Text` の可読性と上下余白を優先する。
- `RUN-049`
  - WPM グラフの `Block` 枠線は薄い黄色で描画し、グラフ本体の緑系線色と高値部分のオレンジ強調は変更しない。
- `RUN-050`
  - `Menu` 画面は `Start Game with Rhythm` を `Practice Mode` の後に表示する。
- `RUN-051`
  - `Start Game with Rhythm` は `Local` 生成を使用し、Google AI Studio または GroqCloud の設定状態に依存しない。
- `RUN-052`
  - `Config` 画面の Game Settings は `RhythmSpeed` を表示し、値は文字列として編集できる。
- `RUN-053`
  - `RhythmSpeed` の使用値は 1 から 5 に丸める。不正値または空値の場合は 2 として扱う。
- `RUN-054`
  - リズムモードの入力位置は画面左端から3文字目の `^` で示す。
- `RUN-055`
  - リズムモードでは、出題文字が秒間 `RhythmSpeed` 文字ぶん右から左へ移動する。
- `RUN-056`
  - リズムモードの空白は表示間隔を作るために使用し、入力対象文字としては扱わない。
- `RUN-056a`
  - リズムモードの非空白文字は、元文字列を等間隔に並べるのではなく、文字位置と文字値から決定する可変ギャップでレーン上に配置する。
- `RUN-057`
  - リズムモードでは、`^` 位置から左へ抜けた文字を表示しない。
- `RUN-058`
  - リズムモードの入力判定は `^` 位置の前後に許容幅を持ち、内側の許容幅で一致した場合は `Hit`、外側の許容幅で一致した場合は `OK` として扱う。
- `RUN-059`
  - リズムモードで許容幅内に一致する未入力文字がない文字入力は typed と miss を増やし、直近判定を `Miss` とする。
- `RUN-060`
  - リズムモードで許容幅を超えて未入力のまま通過した文字は miss として扱い、直近判定を `Miss` とする。
- `RUN-061`
  - リズムモード中は直近判定の `Hit` / `OK` / `Miss`、Miss 数、Hit+OK 数をリアルタイム表示する。
- `RUN-062`
  - リズムモード中は直近判定の `Hit` / `OK` / `Miss` を `^` 付近にも表示し、ユーザーの入力タイミングで更新する。
- `RUN-063`
  - リズムモード中は `Hit` と `OK` を連続成功として扱い、現在の連続成功数をコンボとして保持する。
- `RUN-064`
  - リズムモード中のコンボ表示は 2 以上から開始し、1 以下では表示しない。
- `RUN-065`
  - リズムモード中のコンボ表示は上部ヘッダーではなく、`^` 付近に `{count} Combo!!` の形式で表示する。
- `RUN-066`
  - リズムモード中は入力 Miss と通過 Miss のどちらでも現在のコンボ数を 0 に戻す。
- `RUN-067`
  - リズムモード完了時の Result 画面は typed、correct、hit、ok、miss、accuracy を表示し、通常モードの WPM、生成元、Timed / Practice 表示とは分離する。

## History

- `HIS-001`
  - 履歴ファイルは優先設定ディレクトリ配下の `history.json` とする。
- `HIS-002`
  - 履歴ファイルが存在しない場合は空履歴として扱う。
- `HIS-003`
  - 壊れた履歴ファイルは警告を返し、アプリは空履歴で継続する。
- `HIS-004`
  - よく間違える文字は正解側文字を集計する。
- `HIS-005`
  - 直近10回の推移は保存順の最新10件を古い順に表示する。

## Sentence Generation

- `GEN-001`
  - `Local` 生成はサンプルテキストから空でない文字列を返す。
- `GEN-002`
  - Google の URL は `API URL` と `Model` から `:generateContent` 付きで組み立てる。
- `GEN-003`
  - Google と Groq は設定不足時に `io::Error` を返す。
- `GEN-004`
  - 応答文字列は ASCII ベースへ正規化し、目標文字数で切り詰める。
- `GEN-005`
  - Google と Groq のプロンプトは、リクエストごとに異なる variation seed と日常的な場面カテゴリを含める。
- `GEN-006`
  - Google と Groq の request body には、短すぎる出力を誘発する API 側の出力トークン上限を追加しない。

## Tooling

- `TOOL-001`
  - `Cargo.toml` の `[lints.clippy]` は Cargo が読み込める形式を維持する。
- `TOOL-002`
  - VS Code の TOML スキーマ検証が Cargo の lint テーブルを誤検出する場合は、manifest の lint 強度を下げずにエディタ検証設定で解消する。
- `TOOL-003`
  - 診断解消後も `cargo metadata --no-deps --format-version 1` で manifest が読めることを確認する。

## Release Automation

- `REL-001`
  - `version-check.yml` は `main` への push で `Cargo.toml` が変更された場合に起動する。
- `REL-002`
  - 現在 version は `cargo metadata --no-deps --format-version 1` から取得する。
- `REL-003`
  - 直前コミットの version と現在 version が同じ場合、検証、ビルド、Release 作成は実行しない。
- `REL-004`
  - version 変更時は Release 前に `make check` と `make build` を実行する。
- `REL-005`
  - OS 別 release build job は成果物を作成し、GitHub Actions artifact として upload する。
- `REL-006`
  - GitHub Release 作成は matrix job ではなく、全成果物を収集する単一 job で実行する。
- `REL-007`
  - Release tag は `v<version>` とする。
- `REL-008`
  - 配布成果物名は version と target triple を含む。
- `REL-009`
  - リリースノートは Markdown ファイルとして生成し、version、変更履歴、成果物一覧を含める。
- `REL-010`
  - `softprops/action-gh-release` は生成済みリリースノートを `body_path` として受け取り、収集済み成果物を添付する。
