# 要件定義 (REQUIREMENTS)

## 概要

`rtyping` プロジェクトにユニットテストを導入し、コードの品質担保と将来的なリファクタリングの安全性確保を行う。

## 目的

1. **品質担保**: 主要なロジック（WPM計算、文生成）が意図通り動作することを保証する。
2. **保守性向上**: テストコードにより仕様を明文化し、修正時のデグレを防ぐ。
3. **Rust学習**: Rustにおける標準的なテスト手法（`cargo test`）を実践する。

## 対象範囲

1. **UseCase層**:
   - `src/usecase/wpm.rs`: 計算ロジックの正確性検証。
   - `src/usecase/generate_sentence.rs`: 文生成の基本的な動作検証（パニックしないこと、結果が空でないこと等）。
2. **Domain層**:
   - `src/domain/entity.rs`: サンプルテキスト読み込みの検証。

## 非機能要件

- **ツール**: Rust標準のテスト機能（`#[test]`, `cargo test`）を使用する。
- **実行**: ローカル環境で `cargo test` が全件パスすること。
- **自動化**: GitHub Actions により `push` と `pull_request` を契機に `cargo test` を自動実行すること。
- **リリース保護**: バージョン変更時のリリース処理では、成果物ビルド前にテストが成功していること。

## 2026-04-05 CLI デフォルト値見直し

### 概要

CLI 起動時の初期状態を静音寄りに変更し、時間制限の既定値を 60 秒として明示する。

### 要件

1. 引数未指定で起動した場合、BGM は再生しないこと。
2. 引数未指定で起動した場合、正答時のタイプ音は再生しないこと。
3. 引数未指定で起動した場合、タイムアップは 60 秒であること。
4. `--sound` 指定時は、BGM とタイプ音の両方を有効化すること。
5. CLI ヘルプと関連ドキュメントは実装と同じ意味になるよう更新すること。

## 2026-04-05 API 文字列生成と設定画面

### 概要

タイピング対象文字列の生成元として外部 API を追加し、タイトル画面から設定を編集できるようにする。

### 要件

1. 文字列生成元として `Google AI Studio` と `Groq` をサポートすること。
2. CLI 引数 `--google` 指定時は Google AI Studio を使用し、`--groq` 指定時は Groq を使用すること。
3. `--google` と `--groq` を同時指定した場合は起動時にエラーとして扱うこと。
4. どちらの引数も指定しない場合は、既存のローカル生成を継続利用できること。
5. API 利用時は `--level` に応じた目標文字数の文字列を生成すること。
6. タイトル画面は `Start Game` と `Config` のメニューを表示し、上下キーで選択、`Enter` キーで決定できること。
7. `Config` 画面では Google AI Studio 用と Groq 用の設定を編集できること。
8. 各プロバイダ設定では `API URL`、`API key`、`Model` を入力できること。
9. 設定ファイルは `~/.config/rtyping/config.json` に保存すること。
10. `API key` は保存時に平文で保持せず、暗号化して保存すること。
11. 保存済み設定は次回起動時に読み出し、Config 画面と API 呼び出しに反映できること。
12. API 通信失敗、認証失敗、未設定時にはアプリが異常終了せず、ユーザーが原因を把握できること。
