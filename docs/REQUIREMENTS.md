# 要件定義 (REQUIREMENTS)

## 概要

`rtyping` は Rust 製の TUI タイピングアプリであり、ローカル文生成と外部 API 文生成、設定保存、結果表示を一連の操作として提供する。

## 機能要件

1. 起動時にタイトルメニューを表示し、`Start Game` と `Config` を選択できること。
2. 文字列生成元として `Local`、`Google AI Studio`、`Groq` をサポートすること。
3. `--google` と `--groq` は排他的であり、未指定時は `Local` を使うこと。
4. `Config` 画面では各プロバイダの `API URL`、`API key`、`Model` を編集できること。
5. 設定は起動時に読み込み、保存時は `API key` を平文で保存しないこと。
6. API 利用時に設定不足、通信失敗、復号失敗が発生してもアプリが異常終了せず、メッセージで状態を説明できること。
7. タイピング中は制限時間を計測し、完了またはタイムアウト時に結果画面へ遷移すること。
8. `--sound` 指定時のみ BGM と正答音を有効化すること。

## 非機能要件

1. 主要なユースケースと設定復元処理は `cargo test` で回帰確認できること。
2. UI 状態遷移とランタイム制御は、初見の保守担当者が数時間で追跡できる粒度に責務分割されていること。
3. `docs/` の記述は現行実装と矛盾しないこと。

## 保守上の前提

1. `src/main.rs` は起動と終了処理を中心に保ち、イベントループや入力処理は別モジュールへ分離する。
2. 設定保存の互換処理や API 生成の失敗系は、挙動をテストで固定する。
3. 外部仕様を変えない内部改善でも、読み手が誤解する古い説明は更新対象とする。
