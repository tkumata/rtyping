# ADR: Typing カーソルを縦棒表示に変更

- Date: 2026-04-13
- Status: Accepted

## Context

Typing 画面では現在入力位置が四角カーソルで示されているが、文字入力の文脈では縦棒カーソルの方が一般的で、出題文字列の視認性も保ちやすい。画面内には WPM 表示と Sparkline もあり、カーソル表現が強すぎると全体の読みやすさを損なう。加えて、現在入力位置の文字の右側にカーソルを置く方が、入力順と表示位置の対応が自然になる。

## Decision

- Typing 画面のカーソル形状を四角から縦棒へ変更する
- カーソル形状の変更は Typing 画面に限定し、Menu、Config、Loading、Result には影響させない
- 出題文字列と WPM 表示の可読性を優先し、カーソルは現在入力位置の補助表示として扱う
- 縦棒カーソルは現在入力位置の文字の右側に配置する

## Consequences

- 入力位置がより一般的な文字入力の見え方になる
- Typing 画面の描画実装でカーソル幅と位置の調整が必要になる
- 右側配置により、文字列の読み順に沿った入力フィードバックを保ちやすくなる
- 他画面の状態遷移や設定保存には影響しない
