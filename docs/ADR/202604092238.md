# ADR 202604092238

## 件名

`timeout=0` の練習モードと strict 入力、`Esc` のメニュー復帰、および `Practice Mode` メニューを追加する

## 状況

- 現在の `Typing` は制限時間を前提にしており、タイムアウトすると `Result` へ遷移する。
- 現在の入力処理は誤入力も `push_char` で入力バッファに蓄積するため、入力位置が先へ進んでしまう。
- `Esc` は `Typing` では結果確定として扱われており、メニューへ戻る手段になっていない。
- タイトルメニューには練習モード専用の開始項目がまだ存在しない。
- ユーザー要望は、時間制限なしで練習できるモード、厳密な入力進行、`Esc` による中断復帰、タイトル画面からの `Practice Mode` 開始である。

## 決定

- `-t 0` または `--timeout 0` を練習モードとして扱い、タイムアウトによる終了を発生させない。
- タイトル画面に `Practice Mode` を追加し、選択時は `Local` 生成かつタイムアウト無効で開始する。
- `Typing` の既定挙動を strict にし、誤入力では入力位置を進めず、正しい文字のみ次へ進める。
- `Typing` 中の `Esc` は `Menu` への復帰として扱い、進行中セッションを中断する。

## 理由

- 練習モードを CLI の既存オプションで表現すれば、UI 変更を最小化できる。
- タイトル画面に専用項目を置くと、CLI 引数を使わない利用者にも練習モードの存在が明確になる。
- strict 判定を入力処理で固定すると、現在の `typed_count` や `incorrects` の定義を大きく崩さずに入力体験を改善できる。
- `Esc` を中断操作として統一すると、`Loading` での取り消しと同様の中断導線になり、操作体系が分かりやすい。

## 影響

- `src/presentation/ui/app/menu.rs` と `src/presentation/ui/render/menu.rs` は `Practice Mode` の選択肢追加に追従する。
- `src/runtime/timer.rs` は `timeout=0` を特殊扱いし、タイムアウト通知を送らない。
- `src/runtime/input/gameplay/typing.rs` は strict 判定と `Esc` の戻り先を変更する。
- `src/presentation/ui/app` は通常モード用の制限時間と、練習モードでの一時的な無制限設定を切り替えられる構造へ拡張する可能性がある。
- `src/presentation/ui/app/typing.rs` は、誤入力を入力位置の進行条件に使わない前提に合わせて入力状態の更新を整理する可能性がある。
- `docs/` は練習モードと strict 判定の仕様を追記する。

## 未決事項

- `Esc` でメニューへ戻ったとき、直前の入力状態を完全破棄するか、セッション再開用に保持するかは今回の対象外とする。
- strict 判定時に正答音を鳴らす条件は現行の正答時のみ継続する前提でよいが、実装時に再確認する。
