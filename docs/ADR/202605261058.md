# ADR: リズムモードを通常 Typing と分離して追加する

## Status

Accepted

## Context

通常 Typing は固定された出題文字列を左から順に入力し、WPM、正確率、ミス数、履歴保存を Timed / Practice の文脈で扱っている。

新しいリズムモードは、画面右から左へ流れる文字を `^` 位置で入力する時間軸ベースのゲームである。入力対象は空白を除く文字だけであり、判定には許容幅が必要になる。成績も typed、correct、miss、accuracy を通常モードとは別に扱う必要がある。

## Decision

- `Start Game with Rhythm` をタイトルメニューに追加する。
- リズムモード開始時の文生成は既存の `Local` 生成経路を使い、外部 API 設定には依存しない。
- `AppState` にリズム用の状態を追加し、通常 Typing の入力判定と WPM 履歴から分離する。
- `GameSettings` に `RhythmSpeed` を追加し、既定値は 2、使用値は 1 から 5 に制限する。
- 空白はリズムの間隔としてのみ扱い、入力対象にしない。
- リズムモードの結果は typed、correct、miss、accuracy を表示し、Timed 履歴には保存しない。

## Consequences

- 通常 Typing の strict 判定と WPM 履歴は変更しない。
- Result 画面は通常モードとリズムモードで表示内容を分岐する。
- 今後リズムモードにスコア、コンボ、タイミング評価を追加する場合も、通常モードの履歴形式とは独立して拡張できる。
