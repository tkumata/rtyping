# ADR: Timed セッションの成績履歴を保存し統計表示する

## 背景

結果画面では現在のセッションの WPM、Accuracy、Misses、Time、最終 WPM グラフを確認できるが、過去成績との比較はできない。継続的な改善を見るには、Timed セッションの自己ベスト、平均、直近推移、頻出ミス文字を保存済み履歴から表示する必要がある。

## 決定

- Timed セッション完了時に `~/.config/rtyping/history.json` へ結果を追記する。
- Practice Mode は履歴保存対象外とする。
- 保存項目は WPM、Accuracy、Miss count、入力時間、生成元、モード、正解側ミス文字とする。
- Result 画面に保存済み履歴の自己ベスト WPM、平均 WPM、平均正確率、直近10回の WPM 推移、頻出ミス文字を表示する。
- Menu に `Stats` を追加し、保存済み履歴の統計サマリを確認できる画面へ遷移する。
- ミス文字は「本来入力すべきだった文字」を集計対象にする。

## 根拠

Practice Mode は時間制限なしで反復練習する用途のため、Timed 成績の推移に混ぜると比較軸が崩れる。Result と Stats で同じ統計集計を使うことで、画面ごとの表示差を最小化し、保存形式と UI の意味を一致させる。

## 影響

- 履歴保存用のドメイン型、保存処理、統計集計処理が追加される。
- `AppState` と `MenuItem` に Stats 導線が追加される。
- Result 画面のレイアウトは既存指標と最終 WPM グラフを優先し、統計領域は画面サイズに応じて縮退する。
- 壊れた `history.json` は起動継続可能な警告として扱う。

## 検証

- Timed セッション完了時に履歴が追加されること。
- Practice Mode 完了時に履歴が追加されないこと。
- 履歴が空でも Result / Stats 画面が表示できること。
- 自己ベスト、平均、直近10回、頻出ミス文字の集計が正しいこと。
- `make check` と `make build` が成功すること。
