# ADR: Target Text ブロックの上下余白を固定する

## Status

Proposed

## Context

Typing 画面の `Target Text` ブロックは、出題本文の上に2行の空行を持つ。ユーザー要求では、本文が表示された後にも2行の空行を維持する必要がある。

この変更は `Target Text` ブロック内の表示余白だけを対象とする。入力判定、WPM グラフ、カーソル形状、カーソル位置の意味、ヘッダー、フッター、Result / Stats 画面は既存動作を維持する。

## Decision

- `Target Text` ブロックは、本文前2行、本文、本文後2行の順で描画する。
- 本文が折り返される場合は、折り返し後の本文全体を本文として扱い、その後に2行の空行を描画する。
- 実装箇所は `src/presentation/ui/render/typing.rs` の `Target Text` ブロック描画に限定する。

## Consequences

- 出題本文の上下余白が視覚的に対称になる。
- 長い出題本文でも、本文末尾の直後に余白が残る。
- 本文開始行を維持するため、既存のカーソル位置計算との整合性を保ちやすい。
