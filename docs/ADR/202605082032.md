# ADR: Config 画面の入力位置カーソルを状態として保持する

## Status

Accepted

## Context

Config 画面の文字列入力欄は、これまで入力と削除が常に末尾へ作用していた。ユーザーが長い API URL、API key、モデル名、ゲーム設定値の途中を修正するには、末尾から削除して再入力する必要がある。

## Decision

- Config 画面の編集状態として、現在フィールド内の入力カーソル位置を `App` に保持する。
- `Left` / `Right` は文字列フィールド内のカーソル移動に使い、`Up` / `Down` / `Tab` は既存通りフィールド移動に使う。
- 文字入力はカーソル位置へ挿入し、`Backspace` はカーソル直前の文字を削除する。
- API key 欄は平文表示せず、実文字数と同じ長さのマスク表示でカーソル位置を示す。
- 設定保存形式と暗号化形式は変更しない。

## Consequences

- Config 画面の途中編集が可能になる。
- API key の秘匿表示を維持したまま、入力位置の視認性を改善できる。
- Config 編集の unit test では末尾固定編集ではなく、カーソル位置に基づく挿入と削除を固定する必要がある。
