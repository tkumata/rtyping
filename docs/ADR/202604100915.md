# ADR: 結果画面へ最終 WPM グラフを表示する

- Date: 2026-04-10 09:15
- Status: Accepted

## Context

現状の `src/presentation/ui/render/typing.rs` は、タイピング中に `ratatui::widgets::Sparkline` で WPM 推移をリアルタイム表示している。一方で `src/presentation/ui/render/result.rs` は数値指標のみを表示しており、プレイ後に最終的な WPM の流れを振り返れない。

既に `App` はセッション単位の `wpm_history` を保持しているため、新しい履歴構造を追加せずとも結果画面で同じデータを再利用できる。

## Decision

結果画面にも `wpm_history` を描画する領域を追加し、タイピング中に更新されていた履歴の最終状態をそのまま Sparkline で固定表示する。

Typing 画面と Result 画面の両方で同一の描画規則を使えるよう、WPM グラフ描画は共有しやすい補助関数へ整理する。

## Consequences

- プレイ中はリアルタイム表示、プレイ後は同じ履歴の最終形を参照できる。
- 履歴データ構造を増やさずに機能追加でき、状態管理の重複を避けられる。
- 結果画面のレイアウト再調整が必要になり、狭い端末での情報量バランスを確認する必要がある。
