# ADR 202604092300

## 件名

`Practice Mode` をタイトルメニューに追加し、`timeout=0` の練習モードを明示する

## 状況

- `-t 0` / `--timeout 0` は既にタイムアップのない練習モードとして定義されている。
- ただしタイトル画面の開始導線には `Practice Mode` がなく、練習用途の入口が CLI に偏っている。
- タイトル画面からも練習モードを選べるようにすると、起動時の操作だけで遊び方を選びやすい。

## 決定

- タイトルメニューに `Practice Mode` を追加する。
- `Practice Mode` は `Local` 生成を使い、`timeout=0` の練習モードとして開始する。
- CLI の `-t 0` / `--timeout 0` は従来どおり練習モードの直接指定として残す。

## 理由

- 同じ練習モードでも、CLI とメニューの両方から入れるほうが初回利用時の導線が分かりやすい。
- `Local` 生成と `timeout=0` の組み合わせに固定すれば、既存の Typing ロジックをほとんど変えずに済む。
- 文字列生成源の選択と練習モードの選択を分けると、開始条件の意図が読みやすい。

## 影響

- `src/presentation/ui/app/menu.rs` と `src/presentation/ui/render/menu.rs` は `Practice Mode` を含む選択肢に更新される。
- `src/runtime/input/menu.rs` は `Practice Mode` 選択時に `Local` 生成と `timeout=0` を組み合わせる。
- `docs/HELP.md` と `docs/DesignDoc-ja.md` はメニューとキー操作の説明を更新する。
- `docs/REQUIREMENTS.md`、`docs/SPECIFICATIONS.md`、`docs/DESIGN.md` は練習モードの開始経路を明文化する。

## 未決事項

- `Practice Mode` 選択時の表示上の残り時間を固定表示にするかは、実装時の UI 判断に委ねる。
- 練習モード中の WPM 表示や終了時遷移は既存の Typing / Result 仕様に従う。
