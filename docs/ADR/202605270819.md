# ADR: リズムモードのコンボ表示を入力位置付近へ追加する

## Status

Accepted

## Context

リズムモードは `Hit`、`OK`、`Miss` の直近判定、Miss 数、Hit+OK 数をリアルタイム表示している。連続成功の勢いを確認するには累計の Hit+OK ではなく、現在継続中の成功回数が必要になる。

コンボは入力タイミングのフィードバックであり、上部ヘッダーよりも `^` 付近に表示するほうが視線移動を少なくできる。

## Decision

- `Hit` と `OK` を連続成功として扱う。
- `Miss` は入力 Miss と通過 Miss のどちらでも現在のコンボを 0 に戻す。
- 現在のコンボ数はリズムセッション状態に保持する。
- コンボ表示は 2 以上から開始し、`^` 付近に `{count} Combo!!` の形式で表示する。
- ヘッダーの既存情報は変更しない。

## Consequences

- 通常 Typing の入力判定、WPM、履歴保存には影響しない。
- リズム描画は直近判定とコンボ表示の両方を `^` 付近へ出すため、狭い幅では描画範囲内に収まる文字だけを表示する。
