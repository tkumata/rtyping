# ADR: Typing 画面へ WPM Sparkline を追加する

## Status

Accepted

## Context

- タイピング中の現在 WPM は数値で表示されているが、変動傾向は把握しづらい。
- 追加情報を表示しても、出題文字列の可読性は維持する必要がある。
- `ratatui` には軽量に推移を描画できる `Sparkline` ウィジェットがある。

## Decision

- `Typing` 画面に WPM 推移専用の小さなグラフ領域を追加し、`ratatui::widgets::Sparkline` で描画する。
- グラフ領域は出題文字列と別レーンに配置し、同一領域へ重ね描きしない。
- WPM 履歴はタイピングセッション単位で保持し、開始時に初期化して終了時に破棄する。
- 画面高や幅が不足する場合でも、出題文字列領域を優先し、重なりを避けるレイアウトを採用する。

## Consequences

- `App` に WPM 履歴を保持する状態が追加される。
- `Typing` 描画はグラフ領域を含むレイアウト分割へ変更される。
- タイマー更新または入力更新に応じて WPM 履歴を更新する必要がある。
- テストではセッション開始時の履歴初期化と、描画レイアウトの重なり回避を確認対象へ加える。
