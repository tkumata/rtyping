# ADR: Typing カーソルの折返し位置ずれを解消する

- Date: 2026-04-13
- Status: Proposed

## Context

Typing 画面では、出題文字列が画面幅に収まらない場合に折り返し表示される。現在のカーソル位置計算は、入力済み文字数と表示幅から単純に座標を求めているため、改行境界をまたぐたびに表示位置が実際の文字列とずれていく。改行のたびにずれが累積すると、入力位置の把握が難しくなり、Typing 画面の可読性が落ちる。

## Decision

- Typing 画面のカーソル位置は、折返し後の表示位置に合わせて算出する
- カーソル座標は、出題文字列の実表示レイアウトと同じ折返し条件を基準にする
- 改行境界をまたいでも、カーソルが現在入力位置の右側に保たれるようにする
- この修正は Typing 画面に限定し、Menu、Config、Loading、Result の表示には影響させない

## Consequences

- 長い出題文字列でも、カーソルが行ごとにずれて見えなくなる
- カーソル計算は文字数の単純換算ではなく、描画時の折返しと整合する必要がある
- 既存の縦棒カーソル方針は維持しつつ、位置だけを補正できる
- 他画面の状態遷移や設定保存には影響しない
