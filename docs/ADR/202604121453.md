# ADR: CLI 引数撤廃と Config 画面への設定統合

- Date: 2026-04-12
- Status: Accepted

## Context

現状 `--timeout` / `--level` / `--freq` / `--sound` / `--google` / `--groq` をコマンドライン引数で受け取っているが、TUI アプリとして CLI 引数は発見しにくく、Config 画面に統合する方がユーザビリティが高い。

## Decision

- `GameSettings` 構造体を `AppConfig` に追加し、全設定を `config.json` に保存する
- `GameSettings` のフィールドは `String` 型で保持し、テキストエディタで直接編集できるようにする
- `clap` 依存を削除し `UiHandler` / `CliArgs` を撤廃する
- BGM サウンドはアプリ再起動後に反映される（`BgmHandler` が起動時のみ初期化されるため）
- キープレス音は `app.typing_sound_enabled()` を都度参照するため即時反映される

## Consequences

- CLI でのスクリプト制御ができなくなる
- 設定が一箇所で管理でき、TUI の操作性が統一される
- `config.json` に `game` セクションが追加されるが、`#[serde(default)]` により旧ファイルも読み込み可能
