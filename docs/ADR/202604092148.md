# ADR 202604092148

## 件名

結果画面の正確率表示を既存カウンタから導出する

## 状況

- 現在の結果画面は `Typed`、`Misses`、`Time`、`WPM` のみを表示している。
- `App` は入力文字数と `incorrects` を保持しており、ミス修正回数や実ミス数を分離した状態は持っていない。
- ユーザー要望は結果画面への正確率追加であり、加えて「ミス修正回数と実ミス数を分けるとより詳細になる」という改善余地が示されている。

## 決定

- 今回は結果画面に `Accuracy` を追加する。
- `Accuracy` は総入力数 `total_typed_count` と `incorrects` から `accuracy = (total_typed_count - incorrects) / total_typed_count * 100` で算出する。
- `total_typed_count = 0` の場合はゼロ除算を避けるため `0.0%` を表示する。
- 今回は `incorrects` の意味を変更せず、ミス修正回数と実ミス数の分離は次段階の改善候補として扱う。

## 理由

- 正確率表示は総入力数とミス数から導出でき、ユーザー価値を小さい変更範囲で追加できる。
- `Backspace` で現在入力長が減っても総入力数は維持することで、負の正確率を防げる。
- ミス修正回数と実ミス数の分離には、`Backspace` を含む入力イベントの意味付けを再設計する必要があり、今回の要求に対して変更範囲が広い。
- まず表示項目を増やし、その後に詳細統計の定義を固める方が保守しやすい。

## 影響

- `src/presentation/ui/render/result.rs` は結果表示項目に `Accuracy` を追加する。
- 必要に応じて正確率計算を再利用可能な関数へ切り出す。
- ドキュメントは結果画面の表示内容と算出式を更新する。

## 未決事項

- ミス修正回数を `Backspace` 回数として扱うか、誤入力の取り消し回数として扱うかは未確定である。
- 実ミス数を「最終的に残った誤字数」とするか、「誤入力イベント数」とするかは未確定である。
