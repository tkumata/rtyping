# ADR: Typing 画面の端末カーソルを隠し、現在文字を強調する

## Context

Typing 画面では現在入力位置を端末カーソルで示していた。しかし `,` と `.` のような細い記号では、端末カーソルが文字に重なることで区別しづらくなる。カーソルを単純に消すだけでは、現在どの文字を入力すべきか判別しづらい。

## Decision

- Typing 画面では出題文字列上へ端末カーソルを表示しない。
- 現在入力すべき文字は `Yellow + Bold` で表示し、下線は付けない。
- 未来の未入力文字は `Gray` で表示する。
- 正答済み文字は `Green`、誤入力位置は `White` on `Red` background を維持する。
- 変更範囲は `src/presentation/ui/render/typing.rs` の描画層に限定し、入力判定、strict 挙動、WPM 履歴、Result / Stats 画面には影響させない。

## Consequences

- 端末カーソルが出題文字を隠さないため、細い記号の視認性が上がる。
- 現在入力すべき文字は本文内のスタイルで示されるため、カーソル非表示でも入力位置を把握できる。
- 旧来の縦棒カーソル座標計算は不要になる。
