# ADR: Cargo lint 設定の TOML スキーマ誤検出をエディタ設定で扱う

## Status

Accepted

## Context

`Cargo.toml` の `[lints.clippy]` は Cargo が読み込める形式だが、VS Code の Even Better TOML によるスキーマ検証でテーブル全体が不正と診断される。

`cargo metadata --no-deps --format-version 1` では manifest を正常に解釈できるため、Cargo 側の設定を削除または弱体化すると lint 方針を損なう。

## Decision

Cargo manifest の `[lints.clippy]` は維持する。

エディタ拡張のスキーマ誤検出は VS Code 設定で扱い、実ビルドや lint の振る舞いを変えない。

## Consequences

- Cargo の lint 強度は維持される。
- TOML 拡張の誤検出はワークスペース設定で抑制される。
- manifest の実効性は `cargo metadata --no-deps --format-version 1` で確認する。
