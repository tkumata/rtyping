# ADR: 外部 API 生成プロンプトの固定化回避

## Context

Google AI Studio と Groq に送るプロンプトが毎回同じ文面のため、同じ設定と同じモデルでは似た文章が返りやすい。API 側の出力トークン上限を追加すると短すぎる出力へ退行するリスクがあるため、文字数制御は既存の正規化処理に残す。

## Decision

- 外部 API 用プロンプトにリクエストごとの variation seed を含める。
- variation seed と合わせて日常的な場面カテゴリも変える。
- Google AI Studio と Groq の request body 形状は既存どおり維持する。
- API 側の出力トークン上限は追加しない。
- 最終的な出題文は既存の `normalize_sentence` で目標文字数以下に切り詰める。

## Consequences

- 同じ設定で開始しても、外部 API へ送る指示文そのものが毎回変わる。
- 短すぎる文章を誘発した出力トークン上限制御を避けられる。
- 完全な文字数一致ではなく、既存仕様どおり目標文字数以下の出題文になる。

## Verification

- 連続生成したプロンプトの variation seed が異なることをテストする。
- プロンプトに目標文字数、多様化指示、場面カテゴリが含まれることをテストする。
- 既存の正規化テストで最終文字列が目標文字数を超えないことを確認する。
