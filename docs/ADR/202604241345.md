# ADR: Typing 画面の WPM グラフを活動ベースで猶予付き 0 化し、ピークをオレンジで強調する

## Status

Accepted

## Context

- Typing 画面の WPM グラフは、現在の実装では入力活動が止まるとすぐに 0 を記録していた。
- そのため短い間隔での一時停止でもグラフが 0 に落ち、タイピングの勢いが読み取りにくい。
- 併せて、グラフの読み取りやすさを少し上げるため、高い WPM 区間を視覚的に区別したい。

## Decision

- Typing 中の WPM 履歴は、入力操作が発生していない区間でも 2 秒の猶予までは直前の WPM 推移を維持し、猶予経過後に 0 を記録する。
- WPM グラフは `SparklineBar` の個別スタイルを使い、履歴内の高いバーをオレンジで強調する。
- Result 画面は同じグラフ描画補助を共有し、表示ルールを Typing と揃える。

## Consequences

- 履歴更新側に、最後の入力時刻と 2 秒の猶予を判定するための状態が必要になる。
- グラフ描画は単一色の `Sparkline` から、バーごとの色分けへ少し複雑になる。
- 無入力時の見た目はより直感的になる一方、従来の累積減衰グラフとは見え方が変わる。
